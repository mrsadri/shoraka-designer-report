# گزارش مقایسهٔ کاندیداهای دیزاینر (محرمانه)

صفحهٔ استاتیک و **رمزنگاری‌شده** برای مقایسهٔ سه کاندیدای دیزاینر. روی GitHub Pages میزبانی می‌شود و دو لایه دسترسی دارد.

> 🔐 **محرمانه.** سایت منتشرشده فقط شامل *ciphertext* است. نه نام‌ها، نه مبالغ، نه نام شرکت‌ها و نه رمزها در سورس عمومی وجود ندارند.

## دو لایهٔ دسترسی

| لایه | رمز | چه می‌بینید |
|---|---|---|
| **رمز اول** | باز کردن صفحه | گزارش به‌صورت **ناشناس** (نام‌ها = «کاندیدای ۱/۲/۳»، مبالغ و نام شرکت‌ها مخفی). دانلود غیرفعال است. |
| **رمز دوم** | خاموش‌کردن حریم خصوصی | گزارش **کامل** (نام‌ها، مبالغ، شرکت‌ها) + فعال‌شدن دانلود رزومه و گزارش مصاحبه. |

- کلید هر لایه با **PBKDF2** از رمز آن لایه ساخته می‌شود؛ محتوای هر لایه با **AES-256-GCM** جدا رمزنگاری شده است.
- نسخهٔ **ناشناس** با رمز اول و نسخهٔ **کامل** + PDFها با رمز دوم رمزنگاری شده‌اند. یعنی با داشتنِ فقط رمز اول، نام‌ها و مبالغ اصلاً در دسترس مرورگر قرار نمی‌گیرند.
- سوییچ «حریم خصوصی» به‌صورت پیش‌فرض **روشن** است؛ برای خاموش‌کردن باید رمز دوم وارد شود.

## ساختار پروژه

```
index.html                 # پوستهٔ ورود (رمز اول)
assets/
  styles.css               # استایل‌ها
  crypto.js                # رمزگشایی دو لایه + سوییچ حریم خصوصی
data/                      # ✅ منتشرشده — فقط ciphertext
  payload.js               # نسخهٔ ناشناس (رمز اول) + نسخهٔ کامل (رمز دوم)
  files/<person>/*.enc      # PDFها (رمزنگاری‌شده با رمز دوم)
build/encrypt.py           # اسکریپت ساخت
src/report.html            # ⛔ متن اصلی گزارش + نگاشت ناشناس‌سازی (git-ignored)
files/<person>/*.pdf       # ⛔ PDF اصلی (git-ignored)
```

**هرگز منتشر نمی‌شود:** `src/report.html`، `files/` و **هیچ رمزی**.

## بازتولید (پس از ویرایش محتوا یا تغییر رمز)

```bash
PASSWORD='<رمز اول>' PASSWORD2='<رمز دوم>' python3 build/encrypt.py
git add -A && git commit -m "rebuild" && git push
```

رمزها فقط از environment خوانده می‌شوند و هرگز داخل کد نوشته نمی‌شوند. نیازمندی: پکیج پایتون `cryptography`.

## اجرای محلی

```bash
python3 -m http.server 8000   # سپس http://localhost:8000
```

## امنیت — نکات

- محتوا واقعاً رمزنگاری شده است؛ حتی با مخزن و URL عمومی، بدون رمز چیزی لو نمی‌رود.
- قوت محافظت به **قوت رمزها** بستگی دارد. رمزها را از کانال امن و جداگانه به اشتراک بگذارید.
- سوییچِ حریم خصوصی یک لایهٔ رمزنگاری واقعی است (نه فقط بصری): دانلود و نسخهٔ کامل بدون رمز دوم قابل دستیابی نیست.
